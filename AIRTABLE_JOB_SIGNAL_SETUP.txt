TGTC FINAL_PASS v0.2 - Airtable migration
========================================

Before running the final-pass pipeline, add these fields to the existing Leads table.
Field names must match exactly.

Existing job-signal fields
--------------------------
1. Job Freshness - Single select: fresh, aging, stale_review, unknown_review
2. Job Age Days - Number (integer)
3. Job URL Status - Single line text or single select
4. Job URL Source - Single line text
5. Job Signal Notes - Long text

New final-decision fields
-------------------------
6. Final Decision - Single select: FINAL_PASS, NEEDS_CHECK
7. Decision Reason - Single line text
8. Secondary Reasons - Long text
9. Official Source - URL
10. Evidence Status - Single line text or single select: PASS, NEEDS_CHECK
11. Firmographics Status - Single line text or single select: PASS
12. Contact Alignment - Single line text or single select: PASS
13. Email Validation - Single line text or single select: PASS
14. Validation Version - Single line text
15. Evidence Bundle - Long text

Required behavior
-----------------
- FINAL_PASS -> Relevance=accept and counts toward the target of 30.
- NEEDS_CHECK -> Relevance=review and never reduces the deficit.
- REJECT, UNVERIFIED, and REROUTE remain in internal evidence artifacts only.
- Aggregator-only postings cannot create Airtable rows.
- Low/risky/accept-all/unknown email states cannot create Airtable rows.
- The approved-lead worker blocks any Approved record that is not a validated FINAL_PASS.

Rollout
-------
1. Add all fields.
2. Keep Instantly campaigns paused.
3. Run validate_setup.py --live.
4. Run the pipeline in shadow mode / dry validation first.
5. Inspect the observability artifact and Airtable rows.
6. Approve production enrollment only after the benchmark and review are accepted.
