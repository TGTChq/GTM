TGTC V3.6 — Unreliable partner-mirror Job URL fix

What changed
- trabajo.org URLs are no longer saved as Airtable Job URL, even when they return HTTP 200.
- The pipeline selects a company careers page, ATS, LinkedIn/Indeed, or another stable publisher when available.
- If Trabajo.org is the only available URL, Job URL is left blank and marked unverified_review rather than keeping a misleading link.
- Other aggregator domains are not automatically banned; this change is intentionally narrow.
- Job URL quality is informational and no longer blocks an explicitly approved lead; Relevance remains the review-routing signal.

Install
1. Copy job_signal.py and repair_airtable_job_urls_v36.py into C:\TGTC\tgtc_pipeline
2. Copy tests\test_core.py into C:\TGTC\tgtc_pipeline\tests
3. Replace files when Windows asks.

Validate
python -m unittest discover -s tests -v

Repair existing Airtable rows
python repair_airtable_job_urls_v36.py

No new Airtable fields are required.
