TGTC PIPELINE V3 PATCH — INSTALLATION (WINDOWS CMD)
===================================================

RECOMMENDED: use the patch ZIP, not the full package, so your existing .env,
API keys, Airtable config, CRM CSV, and data folders remain untouched.

1) Close any running pipeline process.

2) Extract TGTC_v3_patch.zip.

3) Copy every extracted file/folder into:

   C:\TGTC\tgtc_pipeline

   Choose "Replace the files in the destination" when Windows asks.

   The patch DOES NOT contain your .env or crm_companies.csv, so it will not
   overwrite those files.

4) Open CMD and run:

   cd /d C:\TGTC\tgtc_pipeline
   python -m unittest discover -s tests -v

   Expected result:

   Ran 23 tests
   OK

5) Repair the Role Focus that is currently blank in Airtable:

   python repair_airtable_role_focus.py

   The script updates ONLY records whose Role Focus is blank. It does not
   overwrite reviewer-edited Role Focus values and does not change Status.

6) Run the next controlled test:

   python -u run_full_test.py --companies 10 --push-airtable

You should now see progress such as:

   [1/4] Scraping fresh jobs from JSearch...
   [GTM Engineer] Searching JSearch...
   [GTM Engineer] Fetched 30 raw postings

NOTES
-----
- New leads receive a specific Role Focus when the JD contains explicit signals.
- When the JD is too vague, a safe role-level fallback is populated and
  Focus Quality is set to manual_required.
- Job aggregators are filtered using known brands plus conservative generic
  publisher signals. This is not limited to ChatGPT Jobs.
- Metrics now separate hiring-manager identification from usable-email coverage.
